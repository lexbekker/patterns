Фабрика по созданию семейства объектов разных типов, в отличии от ФМ, где возвращался один экземпляр одного типа.
Более расширенный вариант шаблона ФМ.
Объекты внутри фабрики связаны по смыслу или по задаче, не связаны по типу.

Структура: Абстрактный объект - Конкретный объект - Абстрактная фабрика - Конкретная фабрика
Применимость:
    1. Когда БЛ программы работает с разными видами связанных друг с другом продуктов, не завися от конкретных
классов продуктов. АФ скрывает от кода, какие продукты будут созданы. Код работает с ними через общий интерфейс.
    2. В программе уже используется ФМ, но изменения предполагают введение новых типов продуктов.
=======================================================================================================================
Преимущества и недостатки:
    + Гарантирует сочетаемость создаваемых объектов
    + Избавляет код от привязки к конкретным типам объектов
    + Выделяет код создания объектов в одно место, упрощая поддержку кода
    + Упрощает добавление новых объектов в программу
    + Реализует принцип открытости/закрытости
    - Усложняет код множеством дополнительных классов
    - Требует наличия всех типов продуктов в каждой вариации
=======================================================================================================================
Отношения с другими паттернами:
    1. Архитектуры начинаются с ФМ и эволюционируют в АФ, Прототип и Строитель
    2. Строитель строит сложные объекты шаг за шагом, АФ создаёт семейства объектов и возвращаает их сразу же.
    3. АФ реализуется с помощью ФМ, но может быть построена и на Прототипе.
    4. АФ может использоваться вместо Фасада, чтобы скрыть платформо-зависимые классы.
    5. АФ может работать совместно с Мостом.
