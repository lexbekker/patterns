Прототип. Копирование объектов, не вдаваясь в подробности их реализации.
Проблема:
    1. Не каждый объект можно полностью скопировать извне, т.к. часть его состояния может быть приватной.
    2. Копирующий код станет зависимым от классов копируемых объектов, чтобы перебрать все поля объекта, нужно привязаться
к его классу. Из-за этого не сможете копировать объекты, зная только их интерфейсы, но не конкретные классы.

* Паттерн поручает создание копий самим копируемым объектам.
* Вводит общий интерфейс для всеъ объектов, поддерживающих клонирование.
* Метод интерфейса создаёт новый объект текущего класса и копирует в него значения всех полей объекта.

Участники: Интерфейс прототипов - Конкретный прототип - Клиент, создающий копию объекта
Хранилище прототипов - содержит предсозданный набор часто используемых, эталонных, готовых к копированию прототипов. Хеш-таблица.
==================================================================================================================================
Преимущества и недостатки:
    + Позволяет клонировать объекты, не привязываясь к их конкретным классам
    + Меньше повторяющегося кода инициализации объектов
    + Ускоряет создание объектов
    + Альтернатива созданию подклассов для конструирования сложных объектов
    - Сложно клонировать составные объекты, имеющие ссылки на другие объекты.
==================================================================================================================================
Отношения с другими паттернами:
    1. Архитектуры начинаются с ФМ, эволюционируют в сторону АФ, Прототипа и Билдера.
    2. Классы АФ реализуются с помощью ФМ, но могут быть построены и на основе Прототипа
    3. Если Команду нужно копировать перед вставкой в историю выполненных команд, поможет Прототип.
    4. Архитектура, построенная на Компоновщиках и Декораторах может быть улучшена за счёт внедрения Прототипа.
    Он позволяет клонировать сложные структуры объектов, а не собирать их заново.
    5. Прототип не опирается на наследование, но требует сложной процедуры инициализации. ФМ построен на наследовании
    6. Снимок иногда можно заменить Прототипом, если объект, чье состояние требуется сохранить в истории, простой,
не имеет активных ссылок на внешние ресурсы, либо их можно легко восстановить.
    7. АФ, Билдер и Прототип могут быть реализованы с помощью Синглтона.
